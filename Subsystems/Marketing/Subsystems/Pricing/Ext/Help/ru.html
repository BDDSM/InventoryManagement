Настройка правил ценообразования

Для настройки правил ценообразования необходимо сформировать классификатор видов цен, в котором регистрируются все виды цен, которые применяются на предприятии. Для того, чтобы назначить различные виды цен на товары определенных групп, формируется классификатор ценовых групп.

Ведение классификатора ценовых групп

Использование классификатора ценовых групп возможно только при включении функциональной опции - "Использовать ценовые группы".

Список ценовых групп задается в справочнике "Ценовые группы". Справочник "Ценовые группы" вызывается из раздела "Маркетинг".

В справочник "Ценовые группы" вводится информация о всех ценовых группах, используемых на предприятии. Эта информация в дальнейшем используется при определении правил расчета цен товаров, принадлежащих к разным группам цен.

При определении правил расчета можно задать различные наценки по товарам, принадлежащим к разным ценовым группам, по отношению к базовым ценам. В соглашении с клиентом можно указать различные виды цен для различных ценовых групп.

Пример. Товары, которые принадлежат к ценовой группе - "Холодильники", согласно соглашению, будут продаваться по оптовым ценам, а товары, которые принадлежат ценовой группе - "Обувь", согласно соглашению будут продаваться по мелкооптовым ценам. При оформлении документов в рамках данного соглашения для этих товаров будут автоматически подставляться различные цены: для товаров, которые относятся к ценовой группе Холодильники - оптовые цены, а для товаров ценовой группы Обувь- мелкооптовые цены.

Ведение классификатора видов цен

В классификатор "Виды цен" вводится информация о всех видах цен (колонках прайс-листа), которые применяются на предприятии: оптовая, мелкооптовая, дилерская и т.д.

Новый вид цен добавляется в справочник "Виды цен". Справочник открывается из раздела "Маркетинг".

Для каждого вида цены необходимо указать наименование. В качестве наименования можно использовать любое название цены: оптовая, дилерская и т.д.

Идентификатор для вида цены необходимо задавать для того, чтобы в дальнейшем можно было бы использовать данный вид цен в формулах расчета цен. Идентификатор может быть введен вручную или заполнен в соответствии с наименованием цены. Для формирования идентификатора на основании наименования цены, необходимо нажать на стрелку, рядом с полем ввода. После записи вида цены информация об идентификаторе не редактируется.

Для каждого вида цен указывается валюта, в которой будут храниться цены данного вида.

Для каждого вида цены указывается признак включения НДС в цену. Если флаг установлен, то по данному виду цен значения цены должны указываться с учетом НДС. Если флаг не установлен, то цены указываются без учета НДС.

Список видов цен, которые применяются при продаже товаров, можно ограничить. Для тех видов цен, которые не используются при продаже товаров, необходимо снять флаг "Использовать при продаже".

Выбор способа задания цены

Для каждого вида цены необходимо выбрать способ задания цены. Предусмотрены три способа задания цены:

  • Задавать вручную . Цены задаются вручную и могут быть использованы для расчета других цен.
  • Заполнять по данным ИБ. Цены автоматически рассчитываются на основании данных, хранящихся в информационной базе. Например,цена рассчитывается, как максимальная цена конкурента или как себестоимость товара. Эти цены могут быть использованы для расчета других видов цен.
  • Рассчитывать по другим видам цен. Цены рассчитываются по заданным формулам на основании других цен. В качестве базовых цен для расчета могут применяться любые виды цен (заданные вручную, заполненные по данным ИБ, рассчитанные на основании других видов цен).

Настройка правил расчета для способа "Заполнять по данным ИБ"

Данные из информационной базы формируются при выполнении запроса к информационной базе и задаются с помощью схемы компоновки данных (СКД).

Формирование схемы компоновки данных возможно только в режиме «толстого» клиента. Открыть форму для настройки СКД можно из формы вида цены по кнопке "Конструктор схемы компоновки данных".

Схема компоновки данных должна содержать только один набор данных, который должен обязательно содержать следующие поля:

  • Номенклатура, типа СправочникСсылка.Номенклатура.
    - Характеристика, типа
  • СправочникСсылка.ХарактеристикиНоменклатуры.
  •  Упаковка, типа СправочникСсылка.УпаковкиНоменклатуры.
  •  Цена, типа число.
  •  Валюта, типа СправочникСсылка.Валюты


Структуру выводимых группировок и полей настраивать не нужно. Она будет игнорироваться.

Настройка правил расчета для способа "Рассчитывать по другим видам цен"

При указании данного способа расчета в списке "Правила расчета" появляется строка "Все ценовые группы". Для этой строки вводится общая формула расчета, которая будет действовать на все ценовые группы товаров.

Если ценовые группы товаров не используются (в настройках параметров системы отключена опция "Использовать ценовые группы), то табличная часть с указанием ценовых групп в карточке вида цен не показывается.

Формулы расчета можно детализировать до ценовых групп. Добавьте в список ценовых групп те ценовые группы, по которым должны применяться правила, отличные от общих правил расчета и введите для них формулы расчета.

Формула расчета цены может быть введена вручную. При ручном вводе следует соблюдать правила написания выражений, которые предусмотрены для встроенного языка 1С:Предприятие. Для проверки правильности написания выражений следует использовать кнопку "Проверить".

Пример. Дилерская цена рассчитывается как оптовая цена минус 40%. Формула для расчета должна быть написана следующим образом: [Оптовая] * 0.4. В квадратных скобках указывается идентификатор вида цены.

При написании формул можно использовать конструктор формул. Конструктор формул открывается при нажатии на кнопку выбора в поле написания формулы.

В конструкторе формул указываются все те доступные виды цен, которые можно использовать при составлении формул расчета, а также операторы, логические операторы и константы, функции. Используя конструктор формул можно создать формулу, проверить правильность ее написания.

После проверки необходимо нажать на кнопку "Сохранить и закрыть". Информация о формуле будет перенесена в карточку вида цены.

Определение диапазонов конечных цен

При продаже товаров можно назначить диапазон цен, по которым будет осуществляться продажа товаров: назначить максимальную и минимальную цену продажи товаров. Также можно зафиксировать ту максимальную цену закупки, выше которой нельзя закупать товар.

Диапазоны цен продажи и максимальная цена закупки устанавливается для предприятия в целом. Отклонения от диапазона допустимых цен продажи контролируется в момент регистрации соглашений и в момент регистрации документов продажи. Отклонения от максимальных цен закупки контролируются в момент оформления соглашений с поставщиками и в момент оформления документов закупки.

Для настройки диапазонов допустимых цен продажи и закупки необходимо открыть справочник "Виды цен" (раздел "Маркетинг") и нажать на кнопку "Настроить диапазоны допустимых цен": откроется диалоговое окно для ввода информации о видах допустимых цен продажи и закупки.

Из списка зарегистрированных видов цен необходимо выбрать минимальный вид цен продажи. Цены, зарегистрированные по данному виду цен, являются теми минимальными ценами, ниже которых не допускается продавать товар.

Аналогичным образом надо выбрать максимальный вид цен продажи.

Необходимо также выбрать тот вид цен, который будет использоваться в качестве максимально допустимых цен закупки. Эти цены будут контролироваться при оформлении документов, сопровождающих поступление товаров: соглашений с поставщиками, заказов поставщиков, документов поступления.

Установка цен номенклатуры

Рассчитываются и фиксируются цены номенклатуры предприятия в соответствии с видами цен. Установка цен осуществляется по инициативе сотрудников предприятия.

Установка цен

Цены, которые используются на предприятии, регистрируются с помощью документа "Установка цен номенклатуры".

Ввод нового документа "Установка цен номенклатуры"

Новый документ вводится из одноименного списка документов ("Установки цен номенклатуры") в разделе "Маркетинг".

В документе указывается та дата, начиная с которой цены вводятся в действие на предприятии.

На закладке "Виды цен" отображаются все виды цен, которые зарегистрированы в справочнике "Виды цен". Необходимо отметить те виды цен, которые предлагается регистрировать документом "Установка цен номенклатуры". В одном документе можно зарегистрировать цены по нескольким видам. Предусмотрена возможность автоматической установки отметки или снятие отметки у всех видов цен.

В списке "Виды цен" дополнительно ( в нижней части диалогового окна) для каждого вида цены отображается информация о тех видах цен, которые являются зависимыми от данного вида цены (рассчитываются на основании данного вида цены) или которые влияют на расчет цен данного вида (используются в качестве базовых цен для расчета цены).

При установке флага у вида цены в соответствующих графах таблицы "Влияет на выбранные цены" и "Зависит от выбранных цен" в виде рекомендаций пользователю устанавливаются флаги у тех видов цен, которые рекомендуется рассчитывать одновременно с выбранным видом цен. Пользователь может отметить рекомендуемые виды цен, чтобы провести полный перерасчет всех цен.

Для заполнения перечня номенклатурных позиций, по которым будут регистрироваться цены, необходимо перейти на закладку "Установка цен". Для перехода можно использовать кнопку "Перейти к установке цен". В табличной части необходимо заполнить список номенклатурных позиций (товары и услуги), по которым должны регистрироваться значения цен.

Позиции номенклатуры могут быть заполнены путем построчного ввода из справочника "Номенклатура". Для ввода новой позиции номенклатуры используйте Действие - "Добавить" или клавишу "INS".

Предусмотрена возможность заполнения списка всех позиций номенклатуры, принадлежащей одной или нескольким ценовой группе. Для заполнения по ценовой группе используется кнопка "Подобрать". В появившемся списке нужно выделить одну или несколько ценовых групп и нажать кнопку "Выбрать". В табличную часть документа будут заполнены все товары, которые относятся к выделенным ценовым группам.

Если для товаров ведется учет по характеристикам, то цены должны быть зарегистрированы для каждой характеристики позиции номенклатуры. Информация о характеристике вводится в табличную часть документа. При использовании варианта подбора по ценовым группам, информация о характеристиках, которые могут быть назначены для товара, заполняется автоматически.

Если для товара ведется учет в разрезе упаковок, то можно назначить цену за конкретную упаковку товара. Информация об упаковке вводится в табличную часть документа.

Ввод информации о новых ценах

Информация о значении тех цен, которые заполняются вручную, вводится пользователем в табличную часть документа. Значения цен регистрируются в той валюте, которая указана для данного вида цен.

Для автоматического перерасчета всех цен, которые зависят от того вида цен, который вводится вручную, необходимо установить режим работы "Рассчитывать автоматически".

При установке этого режима работы при изменении значения цены в строке позиции номенклатуры будут автоматически рассчитаны все зависимые цены и заполнены цены, которые должны быть рассчитаны по данным информационной базы. В этом случае при изменении вручную значения цены по одной номенклатурной позиции пользователь может проконтролировать значения всех зависимых цен.

Перерасчет зависимых цен и заполнение цен, которые рассчитываются по данным информационной базы, пользователь может инициировать в любой момент времени, выполнив действие "Рассчитать".

Для расчета цен не по всем, а только отмеченным позициям, необходимо выполнить следующие действия:

  • выделить в списке те позиции, по которым надо рассчитать цены;
  • нажать на кнопку "Рассчитать";
  •  в появившемся дмалоговом окне установить флаг "Только выделенные";
  •  нажать на кнопку "Рассчитать".

Аналогичным образом можно рассчитать цены только по тем позициям, по которым цены еще не рассчитаны. В этом случае в диалоговом окне настройки расчета цен необходимо установить флаг "Только незаполненные".

Изменение действующих цен

Предусмотрена возможность корректировки ранее зарегистрированных цен.

Для заполнения информации о действующих ценах используется кнопка "Загрузить действующие цены".

Загружается информация только о тех ценах, для которых установлен способ расчета "Задавать вручную". Действующие цены можно загрузить для всех позиций номенклатуры или только для отмеченных позиций номенклатуры.

При загрузке действующих цен их можно сразу изменить(увеличить или уменьшить на определенный процент) и автоматически рассчитать все зависимые цены.

Для загрузки действующих цен пользователь должен выполнить следующие действия:

  •  нажмите на кнопку "Загрузить действующие цены";
  •  в появившемся диалоговом окне будут отмечены все виды цен, цены по которым можно загрузить;
  •  снимите отметку у тех видов цен, которые не надо загружать;
  •  укажите дату, на которую должны быть заполнены цены (по умолчанию устанавливается текущая дата);
  •  нажмите на кнопку "Загрузить".

Если необходимо загрузить цены только по отмеченным позициям, то необходимо в списке номенклатурных позиций выделить нужные позиции, а в диалоговом окне настройки загрузки цен необходимо дополнительно установить флаг "Только выделенные".

Пользователь может заполнить не все значения цен, а только те по которым цены еще не введены. В этом случае в окне настройки загрузки цен необходимо установить флаг "Только незаполненные".

При загрузке действующих цен можно автоматически пересчитать все зависимые цены в соответствии с новыми измененными действующими ценами. Для того, чтобы перерасчет происходил в момент загрузки действующих цен, в документе должен быть установлен флаг "Рассчитывать автоматически".
Пользователь может иницинировать перерасчет зависимых цен в любой момент времени нажав на кнопку "Рассчитать цены".

Регистрация цен

Для регистрации цен на товары и услуги необходимо провести документ "Установка цен номенклатуры". Фактически при проведении документа вводится в действие прайс-лист по указанным в документе видам цен.

Формирование прайс-листов

Прайс-лист формируется из раздела "Маркетинг". По умолчанию прайс-лист формируется только по тем видам цен, для которых установлен флаг "Использовать при продаже". В качестве даты формирования прайс-листа указывается текущая дата. При необходимости можно сформировать прайс-лист на любую дату и по всем видам цен. Прайс-лист формируется при нажатии на кнопку "Сформировать отчет".

При формировании прайс-листа можно установить различные варианты отбора данных.


Прайс-лист может быть сформирован по определенной ценовой группе. В настройках формирования печатной формы прайс-листа можно выбрать нужную ценовую группу или произвольный список ценовых групп. В состав прайс-листа по умолчанию включаются все виды цен с признаком "Использовать при продаже".

Можно убрать флаг отбора цен, которые используются при продаже, и установить произвольный список тех видов цен, которые должны отображаться в прайс-листе. Можно также установить отбор по группе номенклатуры в соответствии с классификатором или выбрать произвольный список групп по классификатору или произвольный список товаров и услуг. Товары и услуги, для которых цены не рассчитаны хотя бы по одному виду цен, из выбранных для формирования прайс-листа, не включаются в прайс-лист.

Формирование типовых торговых соглашений

Типовое соглашение - это набор типовых условий продаж, который используется при продаже товаров. Количество типовых соглашений не ограничено и зависит от разнообразия тех условий продаж, которые применяются на предприятии.

В каждом типовом соглашении указываются свои условия продаж: вид цен, по которым будут осуществляться продажи, лимит кредита, график оплаты и т.д Типовые условия продаж, зарегистрированные в типовом соглашении, могут быть применены как для всех партнеров торгового предприятия, так и для определенного сегмента партнеров (клиентов): оптовые покупатели, дилеры, дистрибьюторы и т.д.

Те условия которые заданы в типовом соглашении контролируются в момент проведения документов, оформленных в рамках типового соглашения.
Те условия, которые явно не заданы в типовом соглашении, могут в дальнейшем уточняться при оформлении документов в рамках данного соглашения.

Открытие списка типовых соглашений

Список типовых соглашений открывается из раздела "Маркетинг".

Список типовых соглашений может быть открыт из справочника "Сегменты партнеров". Для каждого сегмента партнеров можно посмотреть список зарегистрированных типовых соглашений.

Создание типового соглашения

Новое типовое соглашение создается из списка типовых соглашений (раздел "Маркетинг").

Работа с типовым соглашением

Заполнение типового соглашения

В типовом соглашении регистрируется информация о правилах продажи товаров и услуг: цены, по которым будут продаваться товар или услуги, порядок оплаты, условия кредитования и т.д. Те условия продажи, которые будут зарегистрированы в типовом соглашении являются обязательными и не могут быть изменены.

 Если в типовом соглашении какие-либо условия продажи не определены, то эти условия можно дополнительно заполнить при оформлении документов продажи: индивидуальном соглашении, коммерческом предложении, заказах клиента или документах отгрузки.

Пример. В типовом соглашении не указана организация, от имени которой должны оформляться документы. В этом случае документы могут быть оформлены от имени любой организации, информация об организации уточняется при оформлении документов продажи.

При регистрации соглашения можно указать его наименование (произвольный текст). Наименование соглашения будет отображаться во всех списках и служит для удобной визуализации соглашения и для быстрого построчного набора данных в документах.

Срок действия типового соглашения ограничивается датами, указанными в поле Действует с по . При нажатии на кнопку "Установить интервал" даты начала и окончания действия типового соглашения заполняются автоматически согласно выбранному интервалу. Просроченные типовые соглашения отображаются в списке типовых соглашений специальной пиктограммой. Даты начала и окончания действия влияют на возможность выбора типового соглашения при оформлении документов. В документ может быть выбрано типовое соглашения, дата начала действия которого меньше или равна дате документа, а дата окончания действия - больше или равна дате документа.

В поле "Сегмент партнеров" в типовом соглашении указывается тот сегмент партнеров, для которых действует типовое соглашение. Для выбора доступны все сегменты партнеров, кроме тех, для которых установлен способ формирования "Формировать динамически". Если данное типовое соглашение распространяется на всех партнеров, то в данном поле ничего не заполняется. Предварительно всех партнеров торгового предприятия необходимо разделить на определенные сегменты: дилеры, дистрибьюторы, оптовые покупатели и т.д.

Для типового соглашения можно установить признак регулярности. Регулярным соглашением является то соглашение, заданные условия в котором, выполняются с определенной периодичностью. Для регулярного соглашения устанавливается признак "Регулярное", указывается период (неделя, месяц и т.д.) и указывается количество периодов, в течение которых должны выполняться условия по объему продаж и количеству проданного товара. При указании количества периодов автоматически меняется дата окончания действия типового соглашения.

Объем продаж - это та минимальная сумма, на которую клиент обязан купить товар в соответствии с условиями типового соглашения за определенный период времени. Указать минимальную сумму продаж возможно только в том случае, если типовое соглашение является регулярным. Информация вводится в поле "Сумма". Сумма указывается в той валюте, которая определена для типового соглашения.

Пример. По условиям соглашения партнер в течение квартала каждый месяц должен закупать у торгового предприятия товаров на сумму 10000 рублей. При оформлении такого соглашения указываем сумму 10000 рублей, устанавливаем признак "Регулярное", периодичность - месяц и количество периодов - 3.
Кроме указания суммы закупки товаров в табличной части соглашения можно уточнить список тех товаров (их количество и цену), которые должны быть закуплены в рамках типового соглашения. Общая сумма закупок может быть рассчитана на основании данных о товарах, заполненных в табличной части документа. Для заполнения общей суммы в соответствии с суммой, заполненной по конкретным товарам, используется стрелка в поле "Сумма".

Типовое соглашение может быть оформлено в любой валюте, зарегистрированной в справочнике валют. Документы, связанные с данным типовым соглашением, могут быть оформлены только в той валюте, которая определена в типовом соглашении. Указание валюты в типовом соглашении является обязательным.

Указание организации не является обязательным в типовом соглашении. Если поле "Организация" не заполнено, то тогда документы, оформленные в рамках данного типового соглашения, можно оформлять от имени любой организации, зарегистрированной в справочнике "Организации". Если в рамках данного типового соглашения необходимо оформлять документы только от имени одной организации, то тогда эту организацию следует указать в поле "Организация".

В этом случае при оформлении документов в рамках данного типового соглашения будет происходить жесткий контроль той организации, которая указана в документах, оформленных по данному типовому соглашению. Выбор организации осуществляется в списке выбора. Для быстрого выбора организации в поле ввода можно указать ИНН или краткое наименование организации.

В типовом соглашении указывается тот график оплаты, по которому будет контролироваться оплата товаров и услуг в рамках оформляемого типового соглашения. Предварительно в график оплат вводятся все возможные варианты оплаты, применяемые на торговом предприятии: предоплата, кредит, поэтапная оплата и т.д. На основании этого графика оплаты будет заполняться информация об этапах оплаты в документах продажи: "Заказ клиента", "Реализация товаров и услуг".

В типовом соглашении определяется, предоставляется ли кредитный лимит для клиентов в рамках типового соглашения. Информация о предоставлении кредита вводится на закладке "Условия продаж" в поле "Кредит".

Если кредит, предоставляемый партнерам в рамках типового соглашения ограничен, то в типовом соглашении указывается значение "Ограничен" и указывается та максимальная сумма, на которую может быть предоставлен кредит. Сумма кредита задается в той валюте, которая определена для типового соглашения. Если в рамках типового соглашения партнерам предоставляется неограниченный кредит, то выбирается значение "Не ограничен". Если по условиям типового соглашения партнеры должны работать только на условиях предоплаты, без предоставления кредита, то выбирается значение "Отсутствует".

Максимальный срок отсрочки отгрузки товаров или оказания услуг по заказу, оформленному в рамках типового соглашения, указывается в поле "Срок отгрузки" (закладка "Условие продаж"). Срок отгрузки задается в днях. На основании этого показателя и даты документа, указанной в заказе клиента или документе реализации, рассчитывается предполагаемая дата выполнения отгрузки товаров или выполнения услуг по заказу (документу реализации).

Список товаров ограничивается указанным в типовом соглашении сегментом номенклатуры. Если сегмент номенклатуры указан, то в рамках типового соглашения можно будет продавать только те товары, которые относятся к данному сегменту номенклатуры. Если сегмент номенклатуры не указан, то в рамках данного типового соглашения можно продавать все товары. Для выбора доступны все виды сегментов, кроме сегмента номенклатуры, для которого установлен способ формирования "Формировать динамически".

Ставка НДС в типовом соглашении указывается в том случае, если необходимо при оформлении документов в рамках типового соглашения контролировать правильность указания ставки НДС и изменять значение этой ставки в документах продажи не допускается. В типовом соглашении указывается та ставка НДС, которая используется для тех товаров, продажа которых оформляется в рамках данного типового соглашения. Если в соответствии с условиями типового соглашения часть товаров должна продаваться со ставкой, отличной от указанной ставки, то для этих товаров ставка НДС уточняется.

Ставка НДС может быть уточнена как для конкретной номенклатурной позиции, так и для товаров, которые принадлежат к одной ценовой группе. Если для разных организаций используются различные ставки НДС при продаже товаров различных ценовых групп, то оформляется несколько типовых соглашений с указанием конкретной организации и конкретных ставок НДС по ценовым группам товаров.

Признак "Цена включает НДС" указывает на то, какой вариант указания цены выбран для значений цен по номенклатурным позициям в спецификации документа. Если установлен флаг "Цена включает НДС", то цены для номенклатурных позиций указываются с учетом НДС. Если флаг не установлен, то цены должны быть указаны без учета НДС. Установка этого флага также влияет на выбор вида цен в типовом соглашении: для выбора доступны только те виды цен, у которых флаг "Цена включает НДС" совпадает с одноименным флагом в виде цены.

В типовом соглашении можно задать те цены, которые будут заполняться по умолчанию и контролироваться при проведении документов, оформленных по данному типовому соглашению. Цены могут быть заданы для типового соглашения в целом, а также детализированы до ценовых групп номенклатуры или до конкретных позиций номенклатуры.

Если информация о ценах не зарегистрирована в типовом соглашении, то продажа товаров может осуществляться по любым ценам. Цены, по которым будут оформляться продажи, задаются непосредственно при оформлении документов продажи: заказ клиента, реализация товаров и услуг.

В поле "Вид цен" в шапке документа (закладка "Условия продажи") указывается тот общий вид цены, который будет действовать для всех номенклатурных позиций, для которых не указаны иные виды цен, детализированные до ценовых групп или номенклатурных позиций в торговом соглашении.

В типовом соглашении можно определить те ценовые группы, по которым назначаются различные виды цен. Это позволит оформлять продажу товаров, относящихся к разным ценовым группам, по различным ценам в рамках одного типового соглашения. Информация о ценовых группах и видах цен для каждой ценовой группы вводится на закладке "Условия продаж".

Пример. В торговом предприятии создано две ценовые группы : Холодильники и Обувь. В типовом соглашении для ценовой группы "Холодильники" указан вид цен - оптовые, а для ценовой группы "Обувь" - мелкооптовые. Продажа товаров, которые относятся к ценовой группе "Холодильники", будет осуществляться по оптовым ценам, а продажа товаров, которые относятся к ценовой группе "Обувь" - по мелкооптовым ценам.

Для клиентов, которые работают в рамках типового соглашения можно предусмотреть возможность посещения их торговыми представителями. Для этого необходимо установить флаг "Используется при работе с торговыми представителями". Данная информация используется при формировании заданий торговым представителям.

Заполнение спецификации типового соглашения

В табличную часть документа добавляется новая номенклатурная позиция. Новая номенклатурная позиция выбирается из справочника "Номенклатура". В типовом соглашении можно конкретизировать условия для товаров с определенными характеристиками и/или упаковками. В этом случае для товара дополнительно указывается характеристика и/или упаковка товара.

Пример. Товар "Холодильник" продается по фиксированным ценам, зафиксированным в типовом соглашении. Эти цены зависят от объема морозильной камеры, которая в программе оформляется как характеристика товара. В типовое соглашение необходимо ввести несколько строк товара с различными характеристиками и для каждого из них зафиксировать свою цену.

Для каждой номенклатурной позиции может быть указан вид цен. В соответствии с этим видом цен будет рассчитано значение цены в документе продажи, оформленном по типовому соглашению.

Указывается минимальный объем поставок (количество) по каждой номенклатурной позиции. Данная информация заполняется только в том случае, если соглашение является регулярным. Указанное количество является минимальным объемом поставок на каждый период.

Указывается фиксированная цена номенклатурной позиции, которая будет подставляться в документы продаж, созданные в рамках типового соглашения. Если фиксированная цена не указана, то цена будет рассчитываться на основании вида цен, указанного для номенклатурной позиции.

При заполнении документов продажи фиксированная цена, указанная в типовом соглашении для конкретной номенклатурной позиции, имеет более высокий приоритет по сравнению с видом цены номенклатурной позиции, видом цены для ценовой группы и видом цен типового соглашения в целом. Цена указывается в той валюте, которая определена в типовом соглашении.

Расчет цен

После заполнения спецификации типового соглашения для заполнения значений цен по номенклатурным позициям можно вызвать функции расчета цен (Функции - Рассчитать цены).


При заполнении значений цены используется следующий алгоритм:

  • если для номенклатурной позиции задан вид цен, то значение цены заполняется в соответствии с этим видом цены.
  • если вид цен для номенклатурной позиции не задан, то значение цены заполняется в соответствии с тем видом цен, который указан для ценовой группы, в которую входит данный товар.
  •  если вид цен не задан ни для конкретной позиции, ни для ценовой группы, то значение цены заполняется в соответствии с тем видом цен, который указан типового соглашения в целом.

Функцию расчета цен имеет смысл использовать в том случае, если в типовом соглашении необходимо зафиксировать конкретные значения цен, действующие в данный момент времени. При оформлении документов продажи будут заполняться и контролироваться эти значения цен, независимо от того, как изменились значения цены данного вида для торгового предприятия.

Пример. На текущую дату с помощью документа "Установка цен номенклатуры" зафиксированы цены на товар "Кроссовки" по виду цен "Оптовые". Оформлено типовое соглашение со сроком действия, равным текущему кварталу, в котором рассчитаны и зафиксированы цены на товар "Кроссовки". Эти цены будут устанавливаться во всех документах продажи, оформленных в период действия типового соглашения, независимо от того какие новые цены будут введены на предприятии (зарегистрированы документом "Установка цен номенклатуры") в течение текущего квартала.

Регистрация типового соглашения

Для того, чтобы зарегистрировать условия, указанные в типовом соглашении, документ необходимо записать и провести.

Закрытие типового соглашения

Для закрытия соглашения необходимо установить в нем флажок "Закрыто" и провести документ. Закрытые и просроченные соглашения не участвуют при оформлении хозяйственных операций (недоступны для выбора в соответствующих документах).

Для того чтобы закрыть несколько типовых соглашений, необходимо выделить в списке типовых соглашений нужные соглашения и выполнить команду "Функции" - "Закрыть".

Работа со списком типовых соглашений

Для того чтобы в списке не показывались закрытые типовые соглашения необходимо установить флаг "Только открытые".

Для того чтобы в списке показывались только те соглашения, в которых пользователь установлен в качестве ответственного, надо установить флаг "Только мои".
