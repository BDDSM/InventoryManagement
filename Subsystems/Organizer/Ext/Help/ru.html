Настройка наборов дополнительных свойств

Формируются наборы дополнительных свойств для объектов конфигурации - документов и справочников. Любое свойство может быть предназначено для одного или нескольких наборов свойств.
Для ряда справочников справочников поддерживается возможность задавать отдельные наборы свойств для разных подмножеств элементов.

Для каждого свойства определяется:
- Вид свойства (дополнительный реквизит или дополнительное сведение)
- Тип значения (булево, числовое, элемент справочника...)
- Использование в отчетах (используется или нет)
- Порядок отображения в списке.

Формирование и поддержка списка дополнительных свойств

Определяется общий список тех свойств, которые будут использованы на предприятии. При разработке программы в данный список уже добавлены предопределенные свойства, которые используются для решения определенных задач и не могут быть изменены пользователем.
Предопределенные свойства помечаются в списке определенной пиктограммой.
Пользователь может добавить любое количество свойств, которые будут использоваться для анализа данных на его предприятии. Для свойства обязательно определяется его тип: дополнительный реквизит или дополнительные сведения. В зависимости от типа свойства информация будет храниться либо в самом объекте (дополнительный реквизит) или в регистре сведений (дополнительные сведения). И тот и другой тип свойства можно использовать для анализа данных в отчетах. Для каждого свойства определяется тот тип значения (дата, число, справочник и т.д), который будет использоваться при заполнении конкретной информации по свойству. Для удобства использования список свойств может быть сгруппирован. Например можно сгруппировать свойства по конкретным объектам: справочникам, документам, видам номенклатуры и т.д.

Общий список свойств открывается из объекта Дополнительные свойства и реквизиты.

В общем списке свойств уже присутствую предопределенные свойства, которые были определены на этапе разработки. Пользователь может добавить любое количество новых свойств.

Добавление нового свойства

Добавляется новая группа свойства.

Добавляется новая карточка свойства.

Вводится наименование свойства. Например, цвет, размер, марка, производитель, срок хранения и т.д.

Выбирается та группа, к которой относится данное свойство.

Устанавливается флажок для того типа, к которому принадлежит свойство. От этого зависит где будет расположена информация о данном свойстве и как будет хранится информация о свойстве. Если установлен флажок Доп.реквизит, то информация о свойстве будет добавлена в форму объекта и будет храниться в качестве реквизита объекта. Если установлен флажок Доп.сведения, то информация о свойстве будет заполняться в отдельной форме Дополнительные сведения и хранится в регистре сведений.

Для каждого свойства указывается тип значения (число, дата, справочник и т.д). Для указания типа значения открывается отдельная форма Редактирование типов значений, в которой флажком отмечается тот тип значения, который должен быть определен для данного свойства. Если в качестве значения свойства должна быть введена произвольная текстовая информация, то выбирается тип значения - Значение свойств объектов.

Пример 1. Введено свойство срок хранения. Для срока хранения необходимо указать тип значения - дата. При указании значения данного свойства пользователь будет вводить конкретную дату.

Пример 2. Введено свойство Производитель. Список производителей товара хранится в справочнике Контрагенты. Поэтому в качестве типа значений следует определить тип значения справочник Контрагенты. При вводе значения данного свойства пользователь будет выбирать производителя из справочника Контрагенты.

Пример 3. Введено свойство Цвет. Информация о цвете должна вводится текстовой строкой (черный, коричневый, белый и т.п.). В качестве типа значения для такого объекта нужно ввести Значение свойств объектов.

Если данное свойство планируется в дальнейшем использовать для анализа данных в отчетах, то устанавливается флаг Включать в отчеты.

Если в программе уже определен список наборов свойств, то при вводе новой карточки свойства можно определить список тех наборов, к которому относится данное свойство.
Для этого надо нажать на надпись Не входит ни в один набор и в появившейся форме отметить флажками те наборы свойств, к которым должно относиться данное свойство.

Редактирование информации о свойстве

Редактирование информации о свойстве производится аналогично добавлению свойства. Пользователь открывает карточку свойства и изменяет необходимую информацию.

Формирование и поддержка наборов свойств объектов

Наборы свойств применяются для того, чтобы задать список тех свойств, которые предназначены конкретному виду объекта. Например, список свойств, который необходим для анализа данных по клиентам отличается от списка тех свойств, которые необходимо использовать при анализе данных по номенклатуре или при анализе документов продажи. Для каждого объекта используется свой набор свойств. Список наборов свойств, которые применяются для объектов, создаются на этапе разработки. Фактически наличие предопределенного набора свойств у объекта означает, что для данного объекта можно применять механизм работы со свойствами. Задачей пользователя является определение списка тех свойств, которые входят в набор свойств для каждого объекта. Предусмотрена возможность автоматического создания различных наборов свойств для каждого вида номенклатуры.

Создание и редактирование набора свойств для справочников и документов

Список дополнительных сведений, который назначен объекту, открывается при нажатии соответствующей пиктограммы в списке или карточке объекта. Список дополнительных сведений можно открыть также из карточки объекта, выбрав действие Дополнительные сведения. В списке показывается список свойств, который назначен для данного объекта.

Для редактирования набора свойств надо открыть карточку набора свойств. Карточку набора свойств можно открыть из карточки объекта, выбрав действие "Изменить дополнительные реквизиты".

Карточку набора свойств можно также открыть из карточки "Дополнительные сведения", нажав на соответствующую пиктограмму или выбрав действие "Изменить доп.реквизиты и сведения".

Диалоговое окно карточки набора свойств разделено на две части. В правой части отображается список всех дополнительных сведений и реквизитов, которые могут быть добавлены в набор. Для того, чтобы пользователь мог отличить в списке, какое свойство будет добавлено в качестве дополнительного реквизита, а какое свойство будет добавлено в качестве дополнительных сведений, применяются соответствующие пиктограммы. В левой части показывается список тех свойств, которые присутствуют в составе набора в качестве дополнительных реквизитов и дополнительных сведений.

Для добавления свойства в состав набора, достаточно установить курсор на нужное свойство в списке Все дополнительные реквизиты и сведения и нажать клавишу ENTER (или два раза щелкнуть мышью). При этом программа сама определяет, как должно быть добавлено свойство: в качестве дополнительного реквизита объекта или в качестве дополнительного сведения по объекту. После добавления свойства в состав набора в общем списке Все дополнительные реквизиты и сведения свойство помечается, как уже добавленное в состав набора (становится недоступным для выбора).

Для добавления нескольких свойств в состав набора, необходимо в списке Все дополнительные реквизиты и сведения выделить необходимые свойства, используя стандартные возможности платформы, и нажать на кнопку Добавить в набор. Программа определит принадлежность свойств к тому или иному типу и автоматически добавит свойства в соответствующие списки Дополнительные реквизиты и Дополнительные сведения.

Для удаления свойства из состава набора необходимо установить курсор на нужное свойство в одном из списков Дополнительные реквизиты или Дополнительные сведения и нажать на кнопку Удалить. Свойство будет удалено из состава набора свойств, в общем списке Все дополнительные реквизиты и сведения это свойство будет опять доступно для добавления в набор свойств.

Находясь в карточке набора свойств пользователь может ввести новое свойство в общий список свойств Все дополнительные реквизиты и сведения. Новое свойство может быть введено путем создания нового или копированием уже существующего свойства. Также пользователь может добавить новую группу свойств. Все эти действия доступны тем пользователям, которые имеют право на добавление новых свойств в общий список свойств.

Для сохранения информации о наборе свойств объекта нажмите на кнопку Записать и закрыть. Список свойств, которые определены, как дополнительные реквизиты, добавится в качестве дополнительных реквизитов в форме объекта (закладка Дополнительные реквизиты). Список свойств, которые определены, как дополнительные сведения добавится в список дополнительных сведений (в соответствующий регистр сведений).

Создание и редактирование набора свойств для вида номенклатуры

Запись о наборе свойств для вида номенклатуры создается автоматически при записи карточки вида номенклатуры. При этом набору свойств присваивается наименование, аналогичное виду номенклатуры. Для этого набора свойств создается список свойств, который используется для анализа данных по данному виду номенклатуры. При вводе информации о новом товаре и указании для него вида номенклатуры, для товара будут добавлены те дополнительные реквизиты и/или дополнительные сведения, которые были определены для данного вида товара.

Пример. Добавляем новый вид товара Холодильники. Для этого вида добавляем следующий набор свойств: Дополнительный реквизит: Марка холодильника и Дополнительные сведения: Производитель. Добавляем новый товар Холодильник Минск, для которого указываем вид номенклатуры Холодильники. Для этого товара в его форме появится новая закладка Доп.реквизиты, в которой будет добавлен реквизит Марка холодильника, а в списке дополнительных сведениях по данному товару появится дополнительное сведение Производитель.

При записи информации о виде номенклатуры (кнопка Записать) автоматически создается набор свойств номенклатуры. Название набора свойств номенклатуры соответствует названию вида номенклатуры.

Карточка вида номенклатуры создается из списка "Виды номенклатуры". В карточке заполняется наименование вида номенклатуры и все необходимые реквизиты. Для записи карточки вида номенклатуры необходимо нажать на кнопку "Записать".

Для формирования списка свойств для данного вида номенклатуры необходимо нажать на кнопку просмотра набора свойств. Откроется карточка набора свойств данного вида номенклатуры. Необходимо добавить в этот набор свойств те свойства, которые будут использоваться для данного вида номенклатуры. Добавление производится аналогично тому, как это было описано при добавлении свойств в наборы свойств для объектов (справочников и документов).

После того, как свойства будут добавлены в набор свойств, необходимо нажать на кнопку Записать и закрыть для регистрации списка свойств для вида номенклатуры. Информация об этих свойствах будет добавляться в виде дополнительных реквизитов и/или дополнительных сведений в карточку той позиции номенклатуры, для который будет указан данный вид номенклатуры.

Ведение дополнительных свойств объекта

При оперативной работе пользователь указывает конкретные значения для каждого свойства. В зависимости от типа свойства (дополнительный реквизит или дополнительные сведения) пользователь вводит данные о значении свойств либо в форме объекта, либо в дополнительной форме Дополнительные сведения.

Список дополнительных реквизитов в форме объекта располагается на закладке "Доп.реквизиты".

Для заполнения значения дополнительного реквизита пользователь нажимает на кнопку выбора рядом с названием реквизита. В зависимости от типа значения, указанного для свойства, открывается соответствующая форма: справочник значений свойств объекта, поле для указания даты, поле ввода числа, справочник клиентов и т.д. Пользователь вводит или выбирает нужное значение из справочника.

Список дополнительных сведений открывается при нажатии на соответствующую пиктограмму или выбора действия Дополнительные сведения из списка или из формы объекта. В списке указаны все те свойства, которые были определены для данного объекта в его наборе свойств.

Для каждого строки, представленной в списке необходимо ввести значения. Порядок ввода значения , также как и при вводе дополнительных реквизитов, зависит от того типа значения, который указан для свойства.
